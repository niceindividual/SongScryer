<task type="auto">
  <name>Task 1: Create render.js — all four screen renderers</name>
  <files>frontend/js/render.js</files>
  <action>
Create frontend/js/render.js. Import QUESTIONS and getQuestionCopy from './questions.js'. No other imports needed.

Get a reference to the app container at module level: const app = document.getElementById('app');

Export a single function: render(state). It switches on state.screen and delegates to the four private renderers.

renderIdentity(state): Sets app.innerHTML to a div.screen.screen-identity containing an h1 "SongScryer", a subtitle paragraph, and two buttons with class btn btn-identity, each with data-action="select-user" and data-user="matt" or "mike". Text: "Matt" and "Mike".

renderCard(state): Sets app.innerHTML to a div containing: (1) a div.user-badge showing state.user and an anchor with data-action="switch-user" labelled "switch", (2) a div.screen.screen-card with h1 "SongScryer" and a button.btn.btn-primary with data-action="draw-card" labelled "Draw a Card".

renderForm(state): Sets app.innerHTML. Structure:
- div.user-badge (same as card screen)
- div.screen.screen-form containing:
  - div.card-prompt with state.card.prompt text (this is static data — safe for innerHTML)
  - div.progress showing "${state.questionIndex + 1} / ${QUESTIONS.length}"
  - div.question containing:
    - label.question-label with the tone-adapted label text from getQuestionCopy(question, state.card.tone)
    - required marker: if question.required, append span.required-mark with text "*"; else append span.optional-mark with text "(optional)"
    - the input rendered by renderQuestionInput(question, placeholder, state.answers[question.key])

renderConfirmation(state): Sets app.innerHTML. Structure:
- div.user-badge (same as other screens)
- div.screen.screen-confirmation containing:
  - h2 "Submitted!"
  - p.submitted-song-title — MUST use escapeHtml on state.lastSubmission.song_title (see XSS note below)
  - button.btn.btn-primary with data-action="draw-another" labelled "Draw another card"

Define a private escapeHtml(str) function: replaces &, <, >, " with their HTML entities. Use it for any user-provided string in innerHTML templates. song_title and artist are user-provided — always escape. card.prompt is static seed data — safe without escaping.

renderQuestionInput(question, placeholder, currentValue): Returns an HTML string (not sets innerHTML). Switch on question.type:
- 'text': input[type=text].question-input with name, placeholder, value (currentValue ?? ''), autocomplete="off" + button.btn.btn-next with data-action="next-question"
- 'textarea': textarea.question-textarea with name, placeholder, rows=4, current text content + button.btn.btn-next with data-action="next-question"
- 'number': input[type=number].question-input with name, placeholder, value, min=1, max=300 + button.btn.btn-next with data-action="next-question"
- 'choice': div.choice-group[role=group] containing buttons.btn.btn-choice each with data-action="answer-choice", data-key, data-value, and class 'selected' if currentValue matches
- 'multi-select': div.multi-select-group with labels containing checkboxes. Each label gets class 'multi-select-option' plus 'selected' if checked. Checkbox disabled if selected count >= question.max AND this option is not selected. Follow with button.btn.btn-next with data-action="next-multi-select" and data-key. The selected array is currentValue ?? [].
- 'rating': div.rating-group[role=group] with span.rating-label-low showing question.lowLabel, then buttons 1-5 with data-action="answer-choice", data-key, data-value (the integer as string in the attribute), class 'selected' if currentValue === n, then span.rating-label-high showing question.highLabel

NOTE: data-value for rating buttons should be the number as a string in the HTML attribute — app.js coerces it back to Number before calling answerQuestion.
