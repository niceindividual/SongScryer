  <name>Task 1: Create state.js — centralized state machine</name>
  <files>frontend/js/state.js</files>
  <action>
Create frontend/js/state.js as a pure ESM module with no DOM imports and no side effects at module level.

Export exactly these named functions (downstream plans import by exact name):

createInitialState(): Returns { screen: 'identity', user: null, card: null, answers: {}, questionIndex: 0, lastSubmission: null, error: null }. Screen union type: 'identity' | 'card' | 'form' | 'confirmation'.

selectUser(state, user): Calls localStorage.setItem('songscryer_user', user), returns { ...state, user, screen: 'card' }.

drawCard(state, cards): Picks random card with Math.floor(Math.random() * cards.length), returns { ...state, card, screen: 'form', questionIndex: 0, answers: {} }.

answerQuestion(state, fieldKey, value): Returns { ...state, answers: { ...state.answers, [fieldKey]: value }, questionIndex: state.questionIndex + 1 }.

submissionSucceeded(state, songTitle): Returns { ...state, screen: 'confirmation', lastSubmission: { song_title: songTitle }, error: null }.

drawAnother(state): Returns { ...state, screen: 'card', card: null, answers: {}, questionIndex: 0, lastSubmission: null, error: null }.

switchUser(state): Calls localStorage.removeItem('songscryer_user'), returns { ...state, screen: 'identity', user: null, card: null, answers: {}, questionIndex: 0 }.

CRITICAL: All transitions spread state into a new object — never mutate state directly. This prevents answers from previous draws bleeding into new sessions.
  </action>
  <verify>Run the dev server and confirm no import errors in the browser console. Alternatively: node --experimental-vm-modules to verify ESM exports exist without DOM errors. Minimal check: grep the file for all 7 export function names.</verify>
  <done>frontend/js/state.js exists, exports all 7 named functions, uses spread for every transition, writes/reads localStorage key 'songscryer_user'.</done>
</task>
