**deploy.sh** (project root):
A bash script that SSHes into the VPS and runs the deployment commands. This is a convenience script -- the user runs it locally.

```bash
#!/bin/bash
# Usage: ./deploy.sh
# Deploys current code to VPS

npm run build

git push origin main

```

**nginx/songer.conf:**
A reference nginx configuration file. This is NOT deployed automatically -- the user copies/adapts it on the VPS. Include clear comments explaining each directive.

```nginx
# Songer app - nginx reverse proxy config
# Copy to /etc/nginx/sites-available/songer.conf and symlink to sites-enabled
# Then: sudo nginx -t && sudo systemctl reload nginx

# This assumes HTTPS is already configured for h.eino.us via certbot.
# If not, run: sudo certbot --nginx -d h.eino.us

location /theyellow/songer/ {
    # Trailing slash on proxy_pass is CRITICAL -- it strips the /theyellow/songer/ prefix
    # so Express receives clean paths like /api/submissions instead of /theyellow/songer/api/submissions
    proxy_pass http://127.0.0.1:3000/;

    proxy_set_header Host $http_host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    # WebSocket support (not needed now but doesn't hurt)
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
}
```

3. How to test (`sudo nginx -t`) and reload (`sudo systemctl reload nginx`)
4. The certbot command if HTTPS is not yet configured
