Create the project from scratch. Initialize with `npm init -y`, then install dependencies:

```bash
npm install express@5 better-sqlite3 helmet compression cors
npm install -D vite concurrently
```

Note: Use Express 5 (v5.2.1+). It has native async error handling -- no try/catch wrappers needed in route handlers.

**package.json scripts:**
- `"dev:server": "node server/index.js"` (or use nodemon if desired)
- `"dev:client": "vite"` (serves frontend with HMR)
- `"dev": "concurrently \"npm run dev:server\" \"npm run dev:client\""` (runs both)
- `"build": "vite build"` (production frontend build)
- `"start": "node server/index.js"` (production start)
- `"seed": "node server/db/seed.js"` (seed cards into database)
- Set `"type": "module"` in package.json for ESM imports

**.gitignore:** Include `node_modules/`, `dist/`, `*.db`, `data/*.db`, `logs/`

**server/db/database.js:**
- Export `initDatabase(dbPath)` function
- Create Database with better-sqlite3
- Set PRAGMAs: `journal_mode = WAL`, `busy_timeout = 5000`, `foreign_keys = ON`
- Implement migration runner: read `.sql` files from `migrations/` directory, sorted by number prefix, execute those with version > current `PRAGMA user_version`, update user_version after each
- Ensure `data/` directory exists before creating DB file (use `fs.mkdirSync` with `recursive: true`)

**server/db/migrations/001-initial.sql:**
Use the exact schema from research (01-RESEARCH.md Code Examples section):
- `cards` table: id INTEGER PRIMARY KEY, prompt TEXT NOT NULL, tone TEXT NOT NULL
- `submissions` table: all columns from research schema including CHECK constraints, submitted_raw TEXT NOT NULL insurance column, FOREIGN KEY to cards
- Use `CREATE TABLE IF NOT EXISTS` for idempotency

Create the Vite frontend configuration and a minimal working page that proves subpath routing and API connectivity work.

**vite.config.js** (project root):
```javascript
import { defineConfig } from 'vite';

export default defineConfig({
  root: 'frontend',
  base: '/theyellow/songer/',
  server: {
    proxy: {
      '/theyellow/songer/api': {
        target: 'http://localhost:3000',
        rewrite: (path) => path.replace(/^\/theyellow\/songer/, '')
      }
    }
  },
  build: {
    outDir: '../dist',
    emptyOutDir: true
  }
});
```

The `base: '/theyellow/songer/'` is CRITICAL -- it ensures all asset URLs (CSS, JS, images) are prefixed with the subpath in production builds. The proxy config allows the Vite dev server to forward API calls to the local Express server during development.

**frontend/index.html:**

**frontend/js/app.js:**
